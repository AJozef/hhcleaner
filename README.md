# hhcleaner

CLI и приложение для очистки hh.ru: удаляет отклики-отказы, чаты-отказы, чаты по архивным вакансиям и старые чаты. Личная утилита для тех, кто давно ищет работу через hh и устал вручную разгребать накопившиеся отказы.

Неофициальный инструмент. С hh.ru никак не связан.

> ⚠️ Платформа: только Windows. Скрипт использует Windows Task Scheduler и WinRT-уведомления; macOS/Linux не поддерживаются.

## Установка

### ✅ Вариант 1: Готовый .exe (для всех)

Никакого Python ставить не нужно.

1. Скачайте `hhcleaner.exe` из [GitHub Releases](https://github.com/AJozef/hhcleaner/releases/latest) (~50 МБ, один файл).
2. **Дважды кликните по нему.** Откроется окно, которое покажет, что будет делать, попросит войти на hh.ru в окне браузера и почистит аккаунт. В конце покажет, что удалилось, и подождёт нажатия Enter - окно само не закроется.

Вход вы выполняете **на самом сайте hh.ru** в окне браузера: программа не видит ваш пароль и нигде его не хранит. На компьютере остаётся только cookie сессии (как при обычном входе в браузере), в `~/.hhcleaner/`.

> При первом запуске Windows может показать синее окно SmartScreen («Windows защитила ваш компьютер») - это обычное предупреждение для новых неподписанных программ. Нажмите **«Подробнее» → «Выполнить в любом случае»**. Подробности и решение частых проблем - в [INSTALLATION.md](INSTALLATION.md).

Браузер для входа берётся системный (Microsoft Edge есть на каждой Windows 10/11, подойдёт и Chrome) - отдельно ничего скачивать не нужно.

### 🛠️ Вариант 2: Из исходников (для разработчиков)

```powershell
git clone https://github.com/AJozef/hhcleaner.git
cd hhcleaner
python -m venv .venv
.venv\Scripts\activate
pip install -e .
hhcleaner --setup
```

Требуется Python 3.9+. Подробнее - в [DEVELOPMENT.md](DEVELOPMENT.md).

## Использование из командной строки

Двойной клик запускает простой визард. Если нужен контроль - те же возможности доступны флагами:

```powershell
hhcleaner                            # все шаги по умолчанию
hhcleaner negotiations               # только отклики-отказы
hhcleaner old-chats --days 30        # только чаты старше 30 дней
hhcleaner --dry-run                  # показать что будет удалено, ничего не трогая
hhcleaner --status                   # статистика по чатам без удаления
hhcleaner --check                    # проверить, рабочая ли сессия
hhcleaner --self-check               # диагностика окружения
hhcleaner --install-schedule         # зарегистрировать еженедельный запуск
hhcleaner --help                     # все опции
```

Доступные шаги: `read-all`, `negotiations`, `chats-rejected`, `archived-vacancy`, `old-chats`. Шаг `chats-rejected` ходит через API, а при его отказе (401/403) автоматически переключается на браузерный метод.

## .env (опционально)

Если положить рядом файл `.env` с `HH_EMAIL` и `HH_PASSWORD`, скрипт будет:
- автоматически заполнять форму при входе (капчу/2FA всё равно нужно пройти руками);
- в плановых `--no-input` прогонах при истёкшей сессии пытаться сделать **тихий headless-перелогин** - без открытия окна. Если в этот момент hh.ru покажет капчу или 2FA, перелогин не справится и придёт системное уведомление «нужен ручной вход».

Подробнее в [.env.example](.env.example).
